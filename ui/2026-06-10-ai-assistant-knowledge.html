Personal Assistant Domains

PERSONAL

User Profile & Preferences Schema

7 categories: identity, preferences (tone/format), per-project living briefs, discrete facts, communication style model, technical decisions, agent state. Retrieved + injected at session start.

NV: hook already injects; gap is one maintained structured profile note.
PERSONAL

The Executive Assistant "Bible"

Travel dossier (seats, loyalty, dietary), meeting prefs (buffers, energy patterns), "personal anticipation" — record every observed preference immediately, act before asked.

NV: save-everything rule applied to a person; per-domain bible notes.
PERSONAL

Personal CRM Contact Schema

Per contact: basics, birthday, context tags, shared interests, family, gifts/debts, interaction log (last spoke, about what), follow-up + "haven't spoken in N months" reminders.

NV: entity-note template turns wiki/entities into a working CRM.
PERSONAL

Life Admin Domains

5 domains: finance (bills, taxes), home (maintenance, insurance), health (check-ups, prescriptions), subscriptions (auto-renewals), documents (ID expiry). All deadline + record work.

NV: one renewals-calendar note aggregating every expiry date.

Business Assistant Domains

BUSINESS

Company Knowledge Base Contents

SOPs (one doc = 80% of role expectations), brand voice guide in modular AI-retrievable sections (voice principles, banned vocab, annotated examples), org chart + escalation paths.

NV: Perpetual Technologies needs this set in Business-Adviser.
BUSINESS

Operations & Compliance Data

CRM pipeline, invoicing, KPIs via live connectors. Compliance calendar: regulatory, commercial (contract notice windows!), finance/vendor, operations. 60-120 day lead times.

NV: compliance-calendar note surfaced by hook as dates approach.
BUSINESS

Customer Support Knowledge

FAQ + decision trees + policies in modular form. If/then escalation rules with full context transfer. Start at 20% volume; targets: 80% automation, 4.0+ CSAT, <25% escalation.

NV: the checklist for anything customer-facing PT ships.
BUSINESS

Meeting Intelligence

Capture 3 things: what was decided, what happens next, who owns it. Tag every point Decision / Action Required / Deferred. Searchable decision archive stops re-litigating.

NV: adopt outcome tags in session notes → queryable open actions.

How the Assistant Should Work

MECHANICS

Memory Architecture (4 types)

Working (context), episodic (past sessions), semantic (facts/prefs), procedural (workflows). Vector for similarity, graph for relations, SQL for auditable facts. Forgetting is a feature.

NV: maps cleanly; missing piece is decay/importance scoring.
MECHANICS

Proactive Patterns

Daily "Your Day Ahead" briefing (pending, slipped, deserves attention). Pattern model: "always wants X before product meetings." GTD daily mini-review + weekly review still apply.

NV: scheduled morning briefing from tasks + todo + renewals.
MECHANICS

Onboarding Intake Questions

Collect once: business info, tool stack, support types, access, workflow cadence. Killer questions: urgent vs non-urgent channel, "what does success look like in 90 days?", KPIs.

NV: template for wiki/clients + seeds the user profile schema.
MECHANICS

Integration Landscape (MCP)

Minimum dozen: email, calendar, tasks, docs, chat, CRM. Then files, video, accounting, payments. Zapier MCP = 9,000 apps; Composio = 20k tools. Native MCP from HubSpot, Zoho, monday.

NV: missing connector categories: accounting, CRM, file storage.

Guardrails

PRIVACY

PII Rules — What NOT to Store

Data minimization (GDPR 5(1)(c)): masked tokens not card numbers, no PHI in logs, no raw PII to third-party APIs. Persistent memory silently becomes an unauthorized data store without retention policies.

NV: save-everything needs a counterweight for third-party personal data.
SECURITY

Credentials & Least Privilege

No long-lived keys: short-lived scoped credentials, runtime injection so agents never hold secrets. Scope per tool/dataset/action. Step-up approval for high-risk actions. Instant revocation.

NV: audit .mcp.json servers for narrowest scopes.

Top Takeaways for NeuralVault

  • The EA "Bible" is the unifying pattern — an assistant's edge is recorded observation + anticipation, which is exactly NV's save-everything rule pointed at a person instead of code.
  • Forgetting is the missing half of memory — NV persists everything at equal weight; 2026 practice adds importance scoring and temporal decay.
  • Decision / Action / Deferred tags in session notes would make the vault queryable for open commitments.
  • One renewals/compliance calendar note (personal + business dates merged) is the highest-leverage new artifact — pure deadline work the hook can surface automatically.
  • Privacy counterweight needed: third-party health/financial/identity details should be summarized or pseudonymized before saving, not stored raw.
