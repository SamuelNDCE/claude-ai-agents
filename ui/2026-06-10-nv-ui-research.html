NeuralVault UI — Professional UX Research

Research synthesis · 2026-06-10 · Sources: Nielsen Norman Group, Apple HIG, Material Design, IxDF, agent-UX industry research, PKM tool analysis (Obsidian / Notion)

1 · What the research says, mapped to NV

ARecognition over recall (NN/g heuristic #6)

People retrieve information far more easily when they can recognize it than when they must recall it. For a memory app this is the core principle: the UI's job is to put cues in front of Samuel so he never has to remember what's in the vault.

  • Search should suggest before you type: recent queries, recently touched notes, and "continue where you left off" in the search overlay.
  • Hover previews on note links and search results (Obsidian's most-loved feature) — see content without committing to navigation.
  • Rich result snippets: show the matched sentence with the hit highlighted, not just the note title. Title-only results force recall of what's inside.

BVisibility of system status (NN/g heuristic #1) — the #1 rule for agent apps

2025–26 agent-UX research converges on five patterns every agent interface needs: planning visibility, tool-use disclosure, memory surfacing, multi-step workflow tracking, and recovery routing.

  • partly builtNV already has notification rows ("Cortex is working a task", indexing progress). Extend to a persistent, glanceable activity feed.
  • Show the plan, not just the spinner: when Cortex delegates, render the step list with live check-offs. Users trust agents they can watch.
  • Memory surfacing: when an agent answers using vault notes, show which notes it pulled (chips under the reply, clickable). This is transparency AND recognition in one.
  • Recovery routing: every failed agent task needs a visible retry/redirect affordance, not silent failure.

CTrust through progressive delegation (NN/g State of UX 2026)

Users burned by premature AI features resist new ones. The fix is progressive delegation: autonomy expands at the pace of the user's own approval history. Start with "Cortex proposes, you approve"; after a streak of accepted actions, offer to auto-approve that action type. The system earns permission through demonstrated reliability.

DPKM tool research (Obsidian vs Notion)

The consensus from tool comparisons: local-first apps win on minimalism, speed, and ownership — the UI should get out of the way of the notes. Patterns that made Obsidian the designers' choice:

  • Command palette (Ctrl+P / Ctrl+K) as the universal entry point — every action reachable by keyboard, fuzzy-matched. The single highest-leverage feature for power users.
  • Quick capture: a global hotkey to dump a thought into the inbox without leaving the current context. Friction at capture time is the #1 reason PKM systems die.
  • Backlinks panel: "what links here" turns every note into a hub. Graph views demo well but research shows they're rarely used for actual navigation — keep the synapse map as ambient/overview, invest in backlinks + search for real retrieval.
  • Speed is a feature: sub-100ms response to input is the Material responsiveness standard; perceived speed is why people describe Obsidian as "better" than feature-richer rivals.

EComplex-application heuristics (NN/g)

  • Progressive disclosure: show core controls, tuck advanced ones behind "more". NV's settings redesign (all 8 providers visible, auto-save) already follows the right instinct — visible state, no Save button to forget.
  • User control & freedom: undo for destructive/bulk actions ("Undo delete" toast) beats confirmation dialogs — confirmations get blindly clicked, undo actually saves data.
  • Flexibility & efficiency: accelerators for experts (keyboard shortcuts on every list, j/k navigation) invisible to novices.
  • Aesthetic & minimalist: every extra element on the dashboard competes with the signal. One primary action per screen (Apple HIG).

3 · Anti-patterns to avoid

Graph view as primary nav

Demos beautifully, navigates poorly. Keep the synapse map as ambient overview; don't make it the retrieval path.

Chat as the only door

Conversational UI is high-friction for repeat tasks. Direct manipulation (click the note, drag the card) must coexist with Cortex.

Notification noise

Agent apps drown users in status. Batch low-priority events; interrupt only for decisions or failures.

Feature sprawl on the dashboard

Notion's lesson in reverse: every widget added dilutes the ones that matter. Dashboard = status + resume-work, nothing else.

Color-only meaning

Status dots (green = key set) need a second cue — icon, label, or tooltip — for accessibility.

Silent agent failure

A task that dies without a visible error + retry destroys trust faster than any bug.

The one-sentence summary: NV wins by making the vault feel visible — surface what's in memory (recognition), surface what agents are doing (transparency), and make everything one keystroke away (command palette). Speed and minimalism are the brand; resist widgets.
